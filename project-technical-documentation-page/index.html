Introduction

Git is a free and open source distributed version control system designed to handle everything from small to very large projects with speed and efficiency.

Git is easy to learn and has a tiny footprint with lightning fast performance. It outclasses SCM tools like Subversion, CVS, Perforce, and ClearCase with features like cheap local branching, convenient staging areas, and multiple workflows.

Install

GitHub provides desktop clients that include a graphical interface for the most common actions in a repository and automatically updates to the Git command line for advanced scenarios:

  • GitHub for Windows
  • GitHub for Mac
  • Git distributions for Linux and POSIX systems are available on the official Git SCM website. Git for all platforms

Configure

Configure user's information for all local repositories

$ git config --global user.name "[name]"

Set the name you want your commit transactions linked to

$ git config --global user.email "[email]"

Set the email you want your commit transactions linked to

$ git config --global color.ui auto

This variable determines the default value for variables such as color.diff and color.grep that control the use of color per command family. Its scope will expand as more commands learn configuration to set a default for the --color option.

Create Repositories

Start a new repository or get an existing URL

$ git init [project-name]

Create a new local repository with a specific name

$ git clone [url]

Download a project and its version history complete

Make Changes

Review editions and create a commit transaction

$ git status

List all new files or changed files to be commited

$ git diff

Shows difference between files to be commit

$ git add [file]

Puts file on stage mode

$ git diff --staged

Shows the difference between selected files and its latest versions

$ git reset [archive]

Deselects the file but preserves its content

$ git commit -m "[description message]"

Permanently writes the snapshot files to version history

Handling Branches

Name a series of commits and combine the full efforts

$ git branch

List all branches in local repository

$ git branch [branch-name]

Create a new branch

$ git checkout [branch-name]

Go to a specific branch and update files

$ git merge [branch]

Combines the history of the specific branch with the current branch

git branch -d [branch-name]

Delete a branch

Refact Files

Change and remove files

$ git rm [file]

Remove the file from the working directory and select it for removal

$ git rm --cached [file]

Removes the file from version control but preserves the file locally

$ git mv [original-file] [renamed-file]

Rename the file and select it for commit

Delete Traking

Delete temporary files and directories

*.log build/ temp-*

A text file called '.gitignore' suppresses versioning accidental files and directories matching patterns specified

$ git ls-files --other --ignored --exclude-standard

List all ignored files in this project

Save Fragments

Archive and restore incomplete changes

$ git stash

Temporarily stores all modified tracked files

$ git stash pop

Restore recent stash files

$ git stash list

Lists all stash change sets

$ git stash drop

Discards the latest change sets in stash

History Review

Browse and inspect the evolution of project files

$ git log

Lists the version history for the current branch

$ git log --follow [file]

Lists the version history for a file, including name changes

$ git diff [first-branch]...[second-branch]

Shows the difference in content between two branches

$ git show [commit]

Returns metadata and content changes for the specified commit

Undo Commits

Erase mistakes and create a replacement history

$ git reset [commit]

Undo all commits after '[commit]', preserving local changes

$ git reset --hard [commit]

Discards all history and changes for the specified commit

Synchronize Changes

Register a repository marker and change version history

$ git fetch [marker]

Download the entire history of a repository marker

$ git merge [marker]/[branch]

Match the branch marker to the local branch

$ git push [alias] [branch]

Send all commits from the local branch to GitHub

$ git pull

Download history and incorporate changes
